    1. Validator 변경
       Required Annotation은 파라미터가 없는 경우 true를 반환함.
       이로서 하나의 VO 객체로 여러 테이블을 사용하는 경우에도 적용시 문제가 발생되지 않도록 수정

       * 파라미터가 없는 경우는 form tag 내에 해당 속성에 해당하는 <input name="속성명" .... />
         tag가 없는 경우임
       * 만약 파라미터 존재 여부를 체크하고자 한다면 NotNull Annotation을 사용해야 함.
       * input radio tag의 경우 필수 조건이라면 기본 항목을 선택하도록 하는 것이 바람직함
       * input checkbox tag의 경우 아무 값도 선택하지 않는 경우 파라미터가 전송되지 않음을 유의
         즉! 선택 값이 없는 경우 null 값으로 설정됨. (파라미터가 없는 것과 동일)

    2. ValidatorFactory 팩토리 및 구조를 변경하면서 좀더 쉽게 사용자 Validator를 추가 할 수 있도록 수정
       - AbstractValidator 클레스를 부모 클레스로 extends 하고 setAnnotation, doValidate 두개의 메소드를 구현
       - marker 역할을 하는 Annotation을 생성하고 아래 Annotation을 설정함. 이중 validator 속성은 
         본 marker에 해당하는 Validator 클레스를 지정해야 함.
         예 : @ValidatorInfo(validatorName = "range", validator = RangeValidator.class)

         validatorName 속성은 jquery.validator.js 와 동일 한 명칭을 사용해야 함.(jquery validator 지원 필요)
         또한 화면단의 validate를 위해 아래 js 파일에 해당 메소드를 작성해야 함
         /resources/openworks/spi/openworks/validate/openworks.validate.js
         네이밍 규칙은 (validatorName = "range" 셈플) validatorName + Validator 형태로 작성 아래 셈플 참조.
         rangeValidator : function() { ... },
